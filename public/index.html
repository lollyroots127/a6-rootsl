Web Servers

This file is named index.html, and it is the designated homepage of the /~rootsl/a1-rootsl/ directory located on the OSU Engineering web server. When a web browser sends a Hypertext Transfer Protocol (HTTP) request for the directory, the Apache web server, the software used by OSU Engineering, maps the request to this file. The majority of web servers are configured to search for an index.html file whenever they receive a request for /, the root directory. On some web servers, other file names are accepted as homepages. For example, Microsoft web servers will also look for default.html files.

The Inspection tools in web browsers such as Firefox and Google Chrome allow users to view requests and responses between the web client and different servers. The network information can differ based on which protocol is used. When viewing the local file version of this webpage, the network information will show three resources: index.html, main.css, and main.js. The general network information for the index.html file shows the Universal Resource Locator (URL), request method, status code, and referrer policy. It shows limited response and request headers as well. When the same file is accessed through a web server, the network information is quite different. An additional favicon.ico file is shown. The general tab includes a remote Internet Protocol (IP) address for where the server is located. The request and response headers are more detailed, showing information on the server, the user-agent, and time and date signatures.

When viewing the network tab for the file hosted on the OSU Engineering web server, four main resources are listed with different status codes. The index.html file and favicon.ico files have 200 status codes, meaning the files were found and retrieved successfully. The main.css and main.jss files are listed with 404 status codes, meaning the files were not found. The only file located in the /~rootsl/a1-rootsl/ directory was index.html. The main.css and main.jss files are linked in the index.html file but are not provided or located on the web server. Upon inspection, the URL for the favicon.ico file is located inside the root directory of the web.engr.oregonstate.edu server. It is a file provided by the parent directory; therefore, it has a success status code.

The URL of this webpage can be broken into four main components: the scheme, the subdomain, the host domain, and the resource. The scheme in this case is https, which means the web browser used a Secure HTTP protocol to access the requested resource. The subdomains are web and engr. These identify what part of the server the request should be sent to. The host domain or server name is oregonstate.edu. The server name identifies which machine, or server, the resource is hosted on. The Domain Name System (DNS) maps the server name to a machine-understandable IP address. Finally, the path to resource is /~rootsl/a1-rootsl/. This identifies where the server should look to find the resource.

Frontend Design

Frontend design refers to the portion of web development focused on the user experience. It involves designing the visual components of an application, including the styling via CSS, the Graphical User Interface (GUI), and any interactive experiences via JavaScript. Any aspect of a program that interfaces with a user involves front-end design. Key considerations are having a consistent theme, an easy-to-understand interface, and an overall design that is interesting and enjoyable. The measure of how well a system uses each of those components is called usability.

The five "E"s of usability are:

Effective
How well are users able to complete any task?
Efficient
Are users able to complete their tasks quickly & easily?
Easy to navigate
Are new and experienced users able to find and remember where they need to go?
Error-free
Are users able to complete their tasks without encountering errors? Are all accessibility concerns addressed?
Enjoyable or engaging
Do users enjoy using the product? Does it appeal to the intended audience?

Page layout tags are block-level elements that help organize content on a page. While they visually function the same by blocking off sections with a newline, they also have functional purposes. The <header> tag is used for the banner at the top of a page, which typically stays consistent throughout the entire site. It will typically include the name of the site and marketing materials such as a logo or slogan. The <nav> tag provides links to other webpages and a search bar. The <main> tag holds the main body of content on a webpage. Inside the <main> element are <section> elements that group content thematically. A <section> typically has a <h1> headline and <article> elements. An <article> typically has a <h1> headline and content over a single topic. Articles often have an ID selector for navigation and styling purposes. Other page layout components include <aside> elements for floating content, <figure> elements for media, and <blockquote> elements for long quotations. Outside the <main> element is the <footer> for copyright and contact information. A final page layout component is a <div>, which is used as a basic divider and a placeholder for dynamic content.

Anchor elements <a> are used to link to various locations. They will include an attribute that provides where the anchor will link to and text in between the opening and closing tags that will describe the link that the user will click. Anchor tags can be used to link to these different types of content:

  1. External content through a URL, such as <a href="https://example.com/">Example</a>, which will display the highlighted text "Example" and link the user to the https://example.com website.
  2. Internal content through ID tags, such as <a href="#article1">Article 1</a>, which will display the highlighted text "Article 1" and link the user to the element on the page with the id article1.
  3. A new page within the site itself through either relative or absolute paths. Since there is another page in this directory called contact.html, it could be reached through <a href="contact.html">Contact</a>.

Optimizing Images

There are six main specifications for optimizing images on the web. First, images should have descriptive and concise files names to assist with search engine optimization (SEO). Search engine bots will use the file names to categorize images for users. Second, file sizes should be as small as possible to improve load times for webpages. This can be done through image compression. Lossy compression makes rough approximations and discards data, which leads to pixelation. An alternative is lossless compression, which loses no data and is reversible. Third, images should be cropped to fit the exact dimensions of space allocated, which also improves load times. Fourth, images should be saved in the most appropriate file format, which is described below. Certain types of file formats are ideal for different purposes. Fifth, images should be served to the proper resolution for different monitors. The previous standard resolution was 72 pixels per inch (ppi), but higher resolutions are now available. A good webpage will provide different sizes to fit different user resolutions. Sixth, images should be saved with an appropriate color mode. RGB is the standard for .PNG, .JPG, .SVG, and .WebP, while Indexed is better for .GIF or .PNG. It is up to the developer to understand these specifications and optimize them for their users.

As mentioned previously, different file formats are appropriate for different purposes, such as for photos or line art. JPG and WebP files are better suited for photography and screenshots. They can both uses millions of different colors and be compressed down to much smaller files sizes. WebP files also allow for transparency with an alpha color channel. SVG, GIF, and PNG files are better suited for line art. Scalable Vector Graphics (SVG) are precise mathematical images that retain quality when enlarged or shrunk. They typically cannot be compressed as much as other file formats. GIF files can have 8-bit transparency, and they are typically used for short video sequences. PNG files have more flexibility in that they are used for screenshots in addition to line art, as well as using RGB or Index color modes. They have transparency through an alpha color channel and can be placed over or under other colors without distortion.

Favicons

To help identify a website or app, browsers and different devices use what is commonly called a favicon, favorites icon, or touch icon. The symbol is typically saved in different file formats and formatted to display in the appropriate size and type depending on the browser or device. Common file formats are .SVG and .PNG, while .ICO is used for older browsers. An online generator can be used to provide the multiple files used. The symbol can show up in the bookmarks bar or as an app icon for smartphones. The favicon for this webpage is my initials with a green background and is provided in 6 different sizes and formats.
